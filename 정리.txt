ch02. node.js란?
1.  특징
- 구글이 출시한 자바스크립트 실행 엔진(자바스크립트 런타임)이다.
- nodejs 서버가 아니고 실행 엔진이고 서버기능을 하는 모듈이 있다.
- 특유의 간결한 문법과 성능 덕분에 인기가 많다.
- 언어(자바스크립트)하나로 프론트엔드/백엔드 모두 가능한다
 
 2. 성능이 좋은 이유
 - nodejs로 만든 서버는 비동기(non-blocking)이기 떄문이다                 /  리로딩 : 다른페이지로 넘어갈때까지 빙글빙글 로딩이 도는것
 
 3. 단점
 - 고성능의 CPU를 필요로하는 nodejs로 구현하기 힘들다
 ex) 이미지변환, 동영상압축, 숫자연산

ch04. 서버 역할을 하는 express 모듈 / 라이브러리 사용 안해도된다 
- nodejs를 사용해서 웹 프레임워크를 만드는 모듈이다

1. port
- 클라이언트에서 오는 요청이 서버로 들어가는 입구

2. localhost
내 컴퓨터 ip를 작성하지 않아도 접속하게 해주는거

3. 요청응답함수
app.get('/', function(req,res){  
  res.send("하이")
})
get : HTTP 메서드
첫번째 매개변수 : 라우팅    
두번쨰 매개변수 : 콜백 함수   // 파일의 경로로 들어가서 응답하는 내용을 말한다

1) HTTP 메서드
- 요청의 목적이나 방식을 어떻게 할건지 알려주는 메서드

2) 라우팅
- 포트안에 접속할수있는 파일들의 경로를 말한다

3) 콜백함수
- 앞에 get이나 post 함수가 실행되고 난 다음에 실행하는 함수
